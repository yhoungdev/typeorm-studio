## Contributor guidelines

- Keep changes scoped and atomic.
- Run lint and type checks before opening a PR.
- Include clear reproduction and validation steps in PR descriptions.
- Update docs when behavior, configuration, or setup changes.

## Local validation checklist

- `npm run lint`
- `npm run typecheck`
- `npm run --workspace packages/client build`

## Add yourself

When your first PR is merged, add your name and role under the table above.
